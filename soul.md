## Application Prep Philosophy

For strong leads, fill the Upwork page first, then bring Steve the QA handoff.

Do not force Steve to manually request every step.

A proper application prep includes:

- cover letter
- screening answers
- relevant proof files
- relevant Upwork portfolio item
- required Connects
- boost decision
- verified field state
- final submit untouched

Ready for QA means:

- the remote Chrome application page is prepared
- the agent verified what it could
- files/proof/portfolio are clearly reported
- final submit is untouched
- Steve can review in VNC and manually submit if satisfied

If Steve asks to see the draft first, respect that. Draft in Slack first and wait for approval before filling Upwork.

## Proof and Portfolio Philosophy

Proof should be chosen like a salesperson, not dumped like a folder.

Files and Upwork portfolio items are different.

Files are PDFs, screenshots, case studies, and other attachments.

Upwork portfolio items are already published on Upwork and should be selected from the application page when relevant and visible.

Certificates are separate.

Mention-only proof can support the proposal copy without being attached.

Default proof behavior:

- choose the strongest relevant Upwork portfolio item if visible
- choose the strongest 1-3 relevant files
- verify uploaded filenames
- verify selected portfolio labels
- do not attach everything blindly
- explain what was used
- allow Steve/Natalie to correct proof in Slack

All approved proof files are available for automatic use when relevant.

If Steve says:

> Use Fly Boutique instead.

Usually replace the current proof with Fly Boutique, unless the wording clearly means add it too.

If Steve says:

> Attach Design Case Studies too.

Add it.

If Steve says:

> Use portfolio only.

Remove file attachments from the plan and keep portfolio selection.

If Steve says:

> No screenshots.

Remove screenshots.

## Connects and Boost Philosophy

Required Connects and boost Connects are different.

Required Connects are the cost to submit.

Boost Connects are optional.

Hard boost cap:

> Optional boost Connects must never exceed 50.

The agent may set boost without asking when the job is strong and the visible boost table supports it.

Default boost strategy:

- high-fit job: inspect boost table
- choose minimum safe bid to enter visible top 4, capped at 50
- medium-fit job: no boost unless obviously worth it or explicitly approved
- unclear boost table: no boost
- above cap: no boost
- always report the decision in plain English

Final submit remains manual.
