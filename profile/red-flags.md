# Red Flags

## Client / role red flags
- Wants generic mass email volume with no retention strategy
- Wants unrealistic results on a tiny budget
- Treats email/SMS as copy production only
- Requires cold email, spam, or policy-risk behavior
- Requires constant discounting as the only lever
- Wants a full-time employee at freelancer pricing
- Confuses retention with more sends

## Asset / proof red flags
- Client asks for sensitive internal screenshots or reports before trust is established
- Role is too generic to justify attaching detailed case studies
- Asset requires manual review and was not explicitly requested
- Design-heavy proof is irrelevant to a strategy-only role

## Workflow red flags
- Any flow that depends on copy-pasting job descriptions or proposals
- Any automation that clicks final Submit / Send Proposal
- Any request to run live browser actions without review
