# Attachment Rules

## Hard rules
- Approved proof files can be attached automatically when relevant.
- Use `requiresManualReview=true` only for runtime uncertainty, such as an Upwork upload rejection, unsupported file type/size, unclear selector UI, too many assets for the page limit, low proof-match confidence, or an explicit Steve/Natalie instruction to ask first.
- Use `do_not_attach` only for assets explicitly removed from the approved proof set.
- Keep file uploads, Upwork portfolio selections, certificates, and mention-only proof separate in planning and reporting.
- If the role is broad or weakly matched, attach only the strongest relevant proof or use mention-only proof rather than dumping every available asset.

## Good auto-attach cases
- Truly Beauty for beauty/skincare/Klaviyo roles
- Lifely for high-AOV / furniture / home goods / premium DTC roles
- Fly Boutique for fashion/apparel or deliverability rescue roles
- Design Case Studies for design/Figma/email design roles
- Endurance Wellness when the brief is about lifecycle foundation or subscription setup
- Approved screenshots/PNGs when performance proof is strategically useful
- Intro/portfolio PDFs when Steve/Natalie asks for broad proof or an intro deck

## Default selection shape
- Select the strongest relevant Upwork portfolio item if visible
- Attach the strongest 1-3 relevant approved files
- Do not attach every available proof file
- If no upload/portfolio proof is available, use mention-only proof honestly

## Browser prep note
Attachment selection should prepare metadata for browser draft preparation only. It must not trigger live submit actions.
